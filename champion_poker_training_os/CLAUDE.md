## ✦ Karpathy Coding Principles (multica-ai/andrej-karpathy-skills)

These 4 rules apply to every change in this repo:

### 1. Think Before Coding
Surface assumptions and ask for clarification BEFORE writing code.
- "Bu screen'de hero seat nerede gözükmeli?" → ask, don't guess.
- If a screenshot is ambiguous, request specifics.

### 2. Simplicity First
Write the minimal code that solves only what was asked.
- No speculative features. No "while I'm here" cleanups.
- If a 10-line patch fixes the issue, don't write 100 lines.

### 3. Surgical Changes
Edit only what's necessary. Don't refactor unrelated code.
- Touch the screen the user reported broken; leave others alone.
- Don't reformat a file just because you opened it.

### 4. Goal-Driven Execution
Define success criteria up-front, verify before claiming done.
- Run `python3 run_audit.py` before saying "fixed".
- Tests must pass: `python3 -m pytest tests/ -q`.

### Multi-agent system
```
UISimulationAgent  ─ Drives every screen as a user, finds UX bugs
MasterAuditorAgent ─ 9 logic checks (catalog / charts / pipeline / self-play)
GTOMasterAgent     ─ Pro-coach 8-section spot analysis
CoachAgent         ─ Quick GTO + math feedback
ReviewAgent        ─ Tags each decision in a played hand
LeakDetectionAgent ─ Aggregates EV loss by (position, pot type, street)
DrillGeneratorAgent─ Builds drill packs from leak fingerprints
PokerPlayingAgent  ─ Self-play hands using BotBrain
AgentOrchestrator  ─ Chains agents for higher workflows
```

### Workflow conventions
- After ANY change → run `python3 run_audit.py`. If it's not all green, fix before committing.
- Hero seat must always have green halo + "★ YOU" badge — don't break this.
- All `fold/call/raise/jam` buttons in any screen use `GtoActionButton` from
  `app/ui/components/action_buttons.py`. Don't create new raw QPushButton(action).
- The drill catalog is generated by `build_full_catalog()` and must stay ≥ 300.
- Position assignment in `game_loop.start_hand`: dealer_idx = BTN seat (not SB).
  HU is special: dealer = SB.
- Sync to user's Desktop folder after every commit:
  `cp <changed files> "$HOME/Desktop/Champion Poker Training OS/<path>"`
